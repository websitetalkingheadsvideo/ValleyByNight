# **00 — Introduction**
**File:** `Art_Bible_00_Introduction.md`  

The introduction establishes the purpose, scope, and unifying aesthetic philosophy of the Valley by Night Art Bible. It explains how the gothic–noir, Phoenix 1994 atmosphere permeates every piece of art, design, and storytelling in the chronicle.  
It also outlines how the ten major sections work together to maintain a consistent visual identity across portraits, cinematics, 3D assets, UI, and narrative-driven art.

---

# **I — Portrait System**
**File:** `Art_Bible_I_[Portrait_System].md`  

This chapter defines the full set of rules for all character portraits shown throughout the Valley by Night project. These 1024×1024 images form the player’s primary emotional connection to NPCs and must express gothic–noir tone, clan identity, subtle psychological tension, and Phoenix 1994 authenticity.  
It details lighting rules, facial expression guidelines, camera angles, color palettes, clan motifs, and complete prompt structures used for generating portrait art.

---

# **II — Cinematic System**
**File:** `Art_Bible_II_[Cinematic_System].md`  

The cinematic system governs all narrative cutscenes, intro sequences, and VO-driven animations. It sets the shot composition standards, pacing rules, noir lighting philosophy, and supernatural subtleties required to maintain tonal continuity.  
This chapter describes lens choices, camera motion, framing, discipline effects, audio layering, and script structures used to craft the signature Valley by Night cinematic look.

---

# **III — Location & Architecture System**
**File:** `Art_Bible_III_[Location_Architecture].md`  

This chapter defines how the world of Valley by Night looks and feels. It merges Phoenix 1994 realism with Gothic Noir mood and Desert Modern architectural language to create a unique, cohesive environment.  
It profiles all major locations—Camarilla Elysium, Giovanni estates, Setite theaters, Anarch sites, Gangrel desert zones, and more—detailing lighting, textures, structure, and environmental storytelling rules.

---

# **IV — 3D Asset System**
**File:** `Art_Bible_IV_[3D_Asset_System].md`  

The 3D system establishes technical requirements and artistic rules for all CC5 → Blender pipelines, PBR texture work, character models, props, vehicles, and environment objects.  
It defines polycount standards, era-appropriate materials, wardrobe aging, shader guidelines, SubD export rules, and the complete cinematic rendering pipeline to ensure 3D assets match the noir tone.

---

# **V — UI & Web Art System**
**File:** `Art_Bible_V_[UI_Web_Art_System].md`  

This chapter defines the full visual identity of Valley by Night’s web interface. It unifies typography, color palettes, shadows, cards, modals, navigation, iconography, and banners under a coherent noir style.  
It also integrates Bootstrap overrides, responsive behavior, and PHP/JS structure so that the entire web UI supports the project’s atmosphere while remaining functional and modern.

---

# **VI — Storyboards & Animatics**
**File:** `Art_Bible_VI_[Storyboards_Animatics].md`  

This chapter codifies the grayscale, charcoal-like style of all 1920×1080 storyboard panels used in cinematic planning. It defines shot notation, timing rules, camera motion expectations, and atmospheric techniques.  
It also details how animatics are assembled, including soundtrack integration, motion simulation, VO timing, and how these previsualizations guide 3D cinematic production.

---

# **VII — Marketing Materials System**
**File:** `Art_Bible_VII_[Marketing_System].md`  

This chapter defines the entire marketing image pipeline—posters, key art, character announcement cards, banners, thumbnails, and chapter reveal pieces. Marketing art is allowed to be slightly more stylized, but must still obey noir purity rules.  
It sets typography standards, contrast levels, moody lighting, composition rules, faction-specific aesthetics, and full prompt templates for promotional imagery.

---

# **VIII — Floorplan & Blueprint System**
**File:** `Art_Bible_VIII_[Floorplan_System].md`  

The blueprint system defines the gold-on-black and parchment architectural style used to represent all havens, Elysium halls, compounds, tunnels, and Phoenix environments.  
It includes line weights, symbols, labels, texture fills, clan and faction motifs, and layout conventions to ensure spatial continuity between narrative descriptions, locations, and cinematics.

---

# **IX — Naming Conventions & Folder Structure**
**File:** `Art_Bible_IX_[Naming_Folder_Structure].md`  

This chapter defines file naming, versioning, and folder hierarchy used across the entire project. This ensures consistent ingestion by Cursor agents, database systems, the website backend, and the rendering pipeline.  
It includes naming conventions for JSON, images, UI assets, 3D assets, storyboards, floorplans, cinematic files, and zipped deliverables.

# **X — Master Index & Integration Layer**
**File:** `Art_Bible_X_[Master_Index_Integration].md`  

The integration layer binds all systems together. It defines the global aesthetic manifesto, negative/positive aesthetic rules, cross-system visual matrix, and unified production pipeline.  
It ensures the entire project maintains continuity of tone, geography, color, lighting, architecture, discipline effects, and supernatural subtlety. This section overrides all others when consistency is at stake.
