# Chronicle Agent (World State Manager)

## Purpose

The **Chronicle Agent** is the authoritative manager of world state for a single *Valley by Night* chronicle.

It is the **only system component permitted to write to the `chronicles` table**.

All other agents, systems, and UI elements must treat the chronicle as **read-only** and interact with it indirectly via events or requests. This ensures consistency, debuggability, and long-term narrative integrity across a playthrough.

---

## Core Responsibility

The Chronicle Agent answers one question:

> **“Given what has happened so far, what is the current state of the world?”**

It enforces the rules that translate events into persistent changes in the chronicle.

---

## Primary Responsibilities

### 1. World State Ownership

The Chronicle Agent is responsible for maintaining:

- Current night and time-of-night phase
- Global stability meters (Masquerade, faction pressure, hunter attention)
- Major plotline tracking (e.g., BSD knowledge state)
- Canonical facts about the world (Prince status, Elysium state, quarantined zones)

No other agent may mutate these values directly.

---

### 2. Processing Time Events

The Chronicle Agent consumes time events generated by the Time Agent.

When a time event is processed, the Chronicle Agent:
- Locks the chronicle row
- Advances time fields (`current_night`, `time_of_night`, `day_of_week`)
- Applies deterministic updates
- Evaluates rule-based triggers
- Logs the change for audit/debugging
- Commits the update atomically

Time passing never alters the world without passing through this agent.

---

### 3. Applying Rule-Driven World Changes

The Chronicle Agent evaluates **chronicle rules**, including but not limited to:

- Scheduled plot escalations (e.g., week-based events)
- Gradual pressure drift (Anarchs, Hunters, Sabbat)
- Masquerade degradation or recovery
- Faction instability caused by prior actions
- Knowledge propagation (rumors → suspicion → confirmation)

These rules are deterministic and based on:
- Current chronicle state
- Historical state
- Explicit events already recorded

The agent does **not** invent story — it enforces consequences.

---

### 4. Knowledge & Rumor State Management

For major plotlines, the Chronicle Agent tracks:
- Whether a threat or faction exists
- What the wider World of Darkness community believes
- How intense or widespread that belief is

Example patterns:
- Unknown → Rumored → Suspected → Confirmed
- Known but actively suppressed information
- Divergence between truth and belief

This allows misinformation, cover-ups, and paranoia to be modeled explicitly.

---

### 5. Auditability & Debugging

Every applied change should be traceable.

The Chronicle Agent is responsible for:
- Writing to an append-only state log
- Recording before/after snapshots or patches
- Linking changes to their triggering events

This allows:
- Post-mortems (“how did the city collapse?”)
- Debugging incorrect escalations
- Future replay or rollback tools

---

## Design Constraints (Intentional)

- ❌ The Chronicle Agent does **not** control time directly  
- ❌ The Chronicle Agent does **not** handle character stats or inventories  
- ❌ The Chronicle Agent does **not** generate encounters or scenes  
- ❌ The Chronicle Agent does **not** contain narrative prose  

It manages **state**, not storytelling.

---

## Relationship to Other Agents

### Time Agent
- Provides time advancement events
- Never mutates world state

### Encounter / Rumor / Influence Agents
- Read chronicle state
- May submit requests or events
- Must not write to `chronicles`

### UI / GM Tools
- Request changes via events
- Never write directly to the chronicle

The Chronicle Agent is the final authority.

---

## Future Responsibilities (Planned, Not Final)

The following areas are expected to expand over time:

- Formal rule definitions for plot escalations
- Difficulty or danger scaling based on world decay
- End-state evaluation (epilogues, outcomes, world grading)
- New Game+ seeding based on prior chronicle outcomes
- Cross-chronicle comparison tools (analytics/debug)

These will be added incrementally as systems stabilize.

---

## Summary

> The Chronicle Agent is the keeper of canon.

It ensures that:
- Time has consequences
- Actions accumulate meaning
- The world remembers what the players did

Everything else may ask, suggest, or request —  
but **only the Chronicle Agent decides what becomes true**.
