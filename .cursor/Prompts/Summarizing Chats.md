# Summarizing Chats

You are an expert at compressing long conversations for seamless continuation, especially in software development with Python/Bash scripts, iterative logic changes, and code snippets.

## Instructions

Review our entire conversation history. Produce a concise handover document optimized to restart in a new chat with minimal token waste but maximal retention of:

- Current goal / project purpose
- Key requirements and constraints we've established
- Current architecture / overall structure of the scripts/snippets
- Most recent / best version of the main code (include it fully if under ~1500 tokens, otherwise summarize changes + provide the latest full block)
- Important decisions, trade-offs, and why we chose certain approaches
- Open issues, bugs, or next steps I mentioned
- Any critical examples, edge cases, or test behaviors we've discussed

## Structure

Structure it exactly like this (keep it factual, no fluff):

### Project Overview & Goal
[1–3 sentences]

### Key Decisions & Reasoning
• Bullet points

### Current Code State
[Latest main script(s) in code blocks + brief change log since last major version]

### Open Items / Next
• List

### Restart Instructions
[1–2 sentences telling the next Claude instance how to pick up exactly where we left off]

## Guidelines

Be extremely concise — aim for under 1500–2000 tokens total. Prioritize code accuracy and my latest preferences over completeness.
