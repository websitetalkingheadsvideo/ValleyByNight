## Critical Git Limitation

### ⚠️ GitHub Blob Size Limit: 100MB

**GitHub enforces a hard 100MB limit on individual file blobs.** Files larger than this will cause push failures.

**Files that exceed this limit:**
- `rag/index/concept_index.json` (~270MB) - **EXCLUDED FROM GIT** (see `.gitignore`)

**Why this matters:**
- Even if you delete a large file in a later commit, **the blob still exists in git history**
- GitHub will reject pushes that include large blobs anywhere in the commit history being pushed
- This requires **rewriting git history** to remove the large blob before pushing

**Solution:**
- Large generated files (like `concept_index.json`) should be:
  1. Added to `.gitignore` to prevent re-adding
  2. Documented as "generated, do not commit"
  3. Regenerated locally as needed

**If you accidentally commit a >100MB file:**
- Use `git filter-branch` or `git filter-repo` to remove it from history
- Force push required: `git push --force-with-lease origin main`
- **Warning**: Force pushing rewrites remote history - coordinate with team

## Patterns & Conventions

### Index Generation
- Indexes are generated by `rag/scripts/build_indexes.py`
- `concept_index.json` is a large semantic index (excluded from git)
- `book_index.json` is smaller and can be committed

### File Size Guidelines
- **Before committing any file >50MB**: Check if it's generated or can be regenerated
- **If >100MB**: MUST be added to `.gitignore` and excluded from git
- **Generated files**: Should generally be in `.gitignore` unless they're small and essential

## JIT Index Hints

```bash
# Find large files before committing
powershell -Command "Get-ChildItem -Recurse -File | Where-Object { $_.Length -gt 50MB } | Sort-Object Length -Descending"

# Check git for large blobs
git rev-list --objects --all | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | awk '/^blob/ {print substr($0,6)}' | sort -k2 -n -r | head -20
```

## Common Gotchas

- **Large generated files**: Always check file size before committing
- **Git history**: Large files in old commits still block pushes
- **Regeneration**: Document how to regenerate excluded files in README
- **Team coordination**: Force pushes affect everyone - communicate before doing history rewrites
